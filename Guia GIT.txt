Git:
1. Terminal --> "Git init" inicializa git en un proyecto
2. Terminal --> Laburo y si quiero registrar un cambio pongo "git add ." , y tenes el cambio temporal hasta que hagas el commit
3. Terminal --> Queres terminar y guardar el cambio como te pinte, vas, pones "git commit -m "nombre de la version" ", y asi creas otra version.
4. Terminal --> Queres ver una version antigua que ya cambiaste, pones "git log" para ver las versiones guardadas Y
despues "git checkout "numero de la version que aparece" para cargar la version que quieras (esto saca todo lo que tengas en el momento, asi que tenes que hacer un commit
para guardar eso, y lo ultimo que guardes se llama master, para volver ahi, usas "git checkout master")

Ramas: Sirven para que cada programador pueda trabajar en el proyecto en simultaneo sin afectar a los demas o probar cosas, porque crea una version de la pagina extra para trabajar,
donde podes guardar los cambios que quieras intentar y ver el resultado sin afectar a la rama principal.

1. Terminal --> "git branch NOMBRE DE RAMA", asi creas una rama
2. Terminal --> Para navegar entre las versiones de las ramas, uso "git checkout NOMBRE DE RAMA", este comando te lleva a la rama, ya fuera del master que es la ultima version, y donde
los cambios que guardes no van a afectar al master, y funca igual que la principal, git add . , git commit -m ...., y asi.
3. Terminal --> Paso opcional, te gusto el cambio que hiciste en la rama, tenes que actualizarlo al master, lo primero que haces es volver al master con "git checkout master" y 
aca pones "git merge NOMBRE DE RAMA", esta accion fusiona la version en la que estes en el momento con la rama que elijas.
4. Terminal --> Si queres borrar alguna rama porque la vida, usas el comando "git branch -D NOMBRE DE RAMA", lo cual la elimina.
